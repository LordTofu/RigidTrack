3  \section intro_sec Introduction
4 
5  Rigid Track is a software that provides, combined with an OptiTrack camera, the pose
6  estimation of one object in three dimensional space. This is achieved with only one camera
7  in combination with reflective markers. Those are attached to the object ought to be tracked.
8  The accuracy in the range of millimeters and the high update rate of 100 Hz enable use cases for
9  fast and agile objects. The main application is navigation for drones that rely on high precision
10  position data. Where GPS is not available, e.g. indoors or due to a lacking GPS receiver, this
11  setup substitutes for it. Another use case is the pure pose logging when the drone does not
12  depend on the position, e.g. when it is remote piloted by hand. While this setup contains one
13  OptiTrack Flex 3 camera, every other model of OptiTrack should work, despite not tested. With
14  better camera models, e.g. the Prime Series, even outdoor usage is possible. When the capabilities
15  are not sufficient please refer to OptiTracks Software Motive. But keep in mind that this solution
16  needs at least 3 cameras as Rigid Track works with only one.

19  \section softInstall_sec Rigid Track Installation
20 
21  Start the RigidTrack_setup.exe from the enclosed SD card and follow the instructions given in the
22  installation assistant. Default parameters like installation directory or shortcuts to be created
23  can be chosen. But normally clicking Next and keeping the default values should be sufficient. When
24  the installation is completed a shortcut in the start menu and the desktop can be used to start
25  Rigid Track. The program is then successfully installed in C:/Program Files (x86)/TU Munich FSD/Rigid Track.

28  \section source_code Source Code
29  The most interesting file for you is main.cpp. It contains the relevant functions for pose estimation.
30  Camera calibration and other functional aspects are also implemented there.
31  The GUI program code is found in RigidTrack.cpp. communication.cpp deals only with communication from main.cpp to
32  the GUI.
